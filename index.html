Parsl: A Parallel Scripting Library for Python

Parsl is a parallel scripting library for Python. It provides a model by which complex workflows can be represented in an intuitive Python-based control application. It facilitates transparent parallel execution of workflow components (apps) on any distributed or parallel computing system.

Why Parsl?

Parsl is parallel: it runs multiple programs concurrently as soon as their inputs are available, reducing the need for complex parallel programming.

Parsl is easy: short, simple scripts can do large-scale work. The same script runs on multicore computers, clusters, grids, clouds, and supercomputers.

Parsl is fast: it can run a million programs, thousands at a time, launching thousands per second.

Parsl is flexible: it can be used in physical sciences, biological sciences, social sciences, humanities, computer science, education.

Parsl is Python: developers write pure Python code with decorators around parallel applciations.

Using Parsl

Parsl allows you to write functions that execute in parallel. Functions can then be linked together with dependencies to create workflows in Python. Parsl wraps python functions into Apps with the @App decorator. Decorated functions execute in parallel when all their inputs are ready.


@App('python', dfk)
def hello ():
    return 'Hello World!'

app_future = hello()

Unlike a regular Python function, when an App is called it returns an AppFuture. Futures act as a proxy to the results or exceptions that the App will produce once it's execution completes.

In addition to Python functions, Parsl can also be used to coordinate parallel execution of external Apps. Common examples are the many science aplications that are invoked from the command line. Parsl manages input/output data using DataFutures.
